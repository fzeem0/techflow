# Mission 9 — System Administration

This mission models administration without requiring host root access.

1. Audit permissions under `$TECHFLOW_HOME/config` and explain the expected
   modes for public configuration and secrets.
2. Create `backups/final-config.tar.gz` and verify its contents.
3. Write a cron expression for a daily 02:30 backup into
   `reports/backup.cron`; do not install it on the host.
4. Record current resource limits from `ulimit -a` in
   `reports/resource-limits.txt`.
5. Run `techflow check` and save its output as `reports/final-health.txt`.
6. Confirm that all actions remained inside the training workspace.

Run `techflow verify 9` for the final validation.
