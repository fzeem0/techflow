# Mission 2 — Filesystem Disaster Recovery

Repair the deliberately damaged training workspace. Do not operate outside
`$TECHFLOW_HOME`.

1. Find the largest file under `$TECHFLOW_HOME`.
2. Compress `logs/old_debug.log` with gzip.
3. Create `backups/logs.tar.gz` containing the logs directory.
4. Change `config/secret.conf` permissions from the unsafe training value to
   owner read/write only (`600`).
5. Replace every `development` value in `config/app.conf` with `production`.
6. Identify, but do not execute, the three mystery files.
7. Write SHA-256 checksums for the config files to `config/CHECKSUMS.txt`.
8. Create a `current` symlink pointing to the `app` directory.

Run `techflow verify 2` when all repairs are complete.
