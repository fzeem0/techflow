# Mission 6 — Network Diagnosis

Diagnose the container network and save the results in
`$TECHFLOW_HOME/reports/mission-6.txt`.

1. List interfaces and routes using `ip address` and `ip route`.
2. Inspect DNS configuration and resolve `example.com` with `getent` and `dig`.
3. Test TCP connectivity with `curl` and `nc`; external access may be disabled
   by the container operator.
4. Start the training app and compare `curl localhost:8888/health` with a failed
   request to an unused local port.
5. Inspect socket state with `ss -tanp`.
6. Record which observations describe the container rather than the host.
