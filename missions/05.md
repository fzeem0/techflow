# Mission 5 — Process Crisis Management

**Difficulty:** Hard · **Estimated time:** 60 minutes

All processes in this mission must belong to your container user.

1. Start the application with `$TECHFLOW_HOME/scripts/start-app`.
2. Read the recorded PID and inspect it with `ps`, `pgrep` and `lsof`.
3. Request `/health` and `/` with curl.
4. Confirm that port 8888 is listening with `ss`.
5. Observe the application log with `tail`.
6. Stop it cleanly with `$TECHFLOW_HOME/scripts/stop-app` and confirm the PID is
   gone. Do not use broad `pkill -f` patterns.

## Verification

While the app is still running, run the verification and then stop it:

```bash
techflow verify 5
$TECHFLOW_HOME/scripts/stop-app
```
